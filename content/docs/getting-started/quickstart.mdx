This guide walks you through creating a new documentation site using this platform.

## Prerequisites

- Node.js 18+ installed
- pnpm (`corepack enable && corepack prepare pnpm@latest --activate`)
- Git
- Docker (for deployment)

## Getting Started

### Option 1: Use as GitHub Template

If this repo is on GitHub, click **"Use this template"** to create a new repository with all the files.

### Option 2: Clone and Customize

```bash
# Clone the repository
git clone https://github.com/your-org/doc-hosting-platform.git my-docs
cd my-docs

# Remove existing git history and start fresh
rm -rf .git
git init

# Install dependencies
pnpm install
```

### Option 3: Fork for Contributing Back

Fork the repository on GitHub if you want to contribute improvements back to the original.

## Project Structure

```
my-docs/
├── content/
│   └── docs/           # Your documentation files (MDX)
│       ├── getting-started/
│       └── guides/
├── src/
│   └── components/     # UI components (don't modify unless customizing)
├── docs.config.json    # Navigation and site config
├── Dockerfile          # Production Docker build
├── docker-compose.yml  # Easy deployment
└── package.json
```

## Configure Your Site

Edit `docs.config.json` to customize your site:

```json
{
  "name": "My Project Docs",
  "description": "Documentation for My Project",
  "navigation": [
    {
      "group": "Getting Started",
      "pages": [
        "getting-started/introduction",
        "getting-started/installation"
      ]
    },
    {
      "group": "API Reference",
      "pages": [
        "api/authentication",
        "api/endpoints"
      ]
    }
  ]
}
```

## Replace Sample Content

<Steps>
  <Step title="Clear sample docs">
    Remove the sample documentation:
    ```bash
    rm -rf content/docs/*
    ```
  </Step>
  <Step title="Create your structure">
    Create directories matching your navigation:
    ```bash
    mkdir -p content/docs/getting-started
    mkdir -p content/docs/api
    ```
  </Step>
  <Step title="Add your first page">
    Create `content/docs/getting-started/introduction.mdx`:
    ```mdx
    ---
    title: Introduction
    description: Welcome to My Project
    ---

    # Welcome

    This is my project documentation.

    <Tip>
      Use MDX components to enhance your docs.
    </Tip>
    ```
  </Step>
  <Step title="Update navigation">
    Ensure `docs.config.json` matches your file structure.
  </Step>
</Steps>

## Run Locally

```bash
pnpm dev
```

Open `http://localhost:3000` to see your docs.

<Note>
  Changes to MDX files hot-reload automatically. Changes to `docs.config.json` require a server restart.
</Note>

## Deploy

When ready, deploy with Docker:

```bash
docker compose up --build -d
```

Your docs are live at `http://localhost:3000`.

## Workflow Summary

| Step | Command | Result |
|------|---------|--------|
| Clone | `git clone ... && cd my-docs` | Get the platform |
| Install | `pnpm install` | Install dependencies |
| Configure | Edit `docs.config.json` | Set up navigation |
| Write | Create MDX files in `content/docs/` | Add your content |
| Preview | `pnpm dev` | Local development |
| Deploy | `docker compose up --build -d` | Production |
