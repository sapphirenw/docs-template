## Columns

Use columns to arrange content side by side.

<Columns cols={2}>
  <Column>
    ### Left Column
    This content appears on the left side. You can include any markdown or components here.
  </Column>
  <Column>
    ### Right Column
    This content appears on the right side. Perfect for comparisons or related information.
  </Column>
</Columns>

## API Documentation Components

### Parameter Fields

Document your API parameters:

<ParamField name="apiKey" type="string" required>
  Your unique API key for authentication.
</ParamField>

<ParamField name="limit" type="number" default="10">
  Maximum number of results to return.
</ParamField>

<ParamField name="offset" type="number" default="0">
  Number of results to skip for pagination.
</ParamField>

### Response Fields

<ResponseField name="id" type="string">
  Unique identifier for the resource.
</ResponseField>

<ResponseField name="created_at" type="timestamp">
  ISO 8601 timestamp of when the resource was created.
</ResponseField>

### Expandable Content

<Expandable title="Show nested properties">
  <ResponseField name="user.name" type="string">
    The user's display name.
  </ResponseField>
  <ResponseField name="user.email" type="string">
    The user's email address.
  </ResponseField>
</Expandable>
